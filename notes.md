
# MemorizeApp プロジェクトノート

## 概要
歌詞・セリフなど任意テキストを暗記するためのWebアプリ

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | HTML構造のみ。CSSとJSを外部参照。画面レイアウト・ボタン定義 |
| `style.css` | 全スタイル定義。テーマ変数（CSS custom properties）・レイアウト・各コンポーネントの見た目 |
| `app.js` | 全ロジック。localStorage操作・モード切り替え・トークン生成・テストモード（入力判定・スコア）・テーマ切り替え |

## 仕様

### フェーズ1（現在）
- ログインなし
- テキストはブラウザの localStorage に保存
- 技術：HTML / CSS / JavaScript（3ファイル構成）

### フェーズ2（将来）
- Google ログイン対応
- Firebase でクラウド保存（どの端末からでもアクセス可能）

## 機能一覧
- テキスト入力・保存・削除・編集
- 隠し方の選択：
  - そのまま（全文表示）
  - 単語隠し（Intl.Segmenter で単語・助詞単位に分割）
  - 一文隠し（文単位で隠す）
  - 頭文字のみ
  - テスト（順番入力 / 自由入力）
- クリック／スペース／Enterで順番開示、Backspace／戻るで巻き戻し
- セクションラベル（[1番][サビ]など）対応
- 目次パネル（タイムライン型、スクロール連動・クリックジャンプ）
- 4種カラーテーマ
- 初回起動時にサンプルテキスト自動挿入

## 感想
### 2026-04-20
- 一文隠しと全隠しボタンの仕様が同じ。この仕様であればどちらかのボタン一つでいい。
- UIが若干使いづらい。具体的には、一文隠しは結構使いやすいのだけれど単語隠しボタンの仕様が一文字一文字文字をクリックする仕様になっているためクリックしづらい。できれば、一文ごとに自然言語処理で本当に単語や接続語ごとに自動で分けて分解できれば使いやすいいかな。
- 将来的に、自分で文字を打って（音声入力して）正解率を出す様な仕様もほしい。その場合はちゃんとフレーズごとに暗記を試したり、その文章まるまるを試したりモードを切り替えられたら嬉しい。
- 全体的UIをもっとわかりやすくしたいかな。最初は登録画面でいいけど、あとは左ペインに登録した文章、歌詞をスレッドとしてタイトルを表示できれば嬉しい。クリックすれば中央メイン画面が移り変わるようなイメージがいい。
- デザインもちょっと安っぽい気がする。シンプルかつ触りたくなる様なデザインがいい。
- 文字送りについてだが、基本メイン画面をクリックすることで「次へ」のボタンとみなした方が使い勝手がいいかな。あと基本はクリックすれば上から順にワードを開示していく形式が使いやすいかも。その場合、２番だけやりたいとか、ここの文章からスタートしたいとかもあるかもしれないので、そこはスタート位置を変更できるようなボタンがあった方が使いやすい。
